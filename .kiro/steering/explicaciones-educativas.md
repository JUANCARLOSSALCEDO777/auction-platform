# Explicaciones Educativas

## Comandos de terminal

Cada vez que se use un comando de terminal **nuevo** (que no se haya explicado antes en la conversación), incluir una breve explicación de qué hace y por qué se usa. No repetir la explicación si el comando ya fue usado anteriormente.

Formato:
```
> comando ejecutado
→ Explicación breve de qué hace
```

## Tecnologías y conceptos

Al implementar o configurar una tecnología, patrón o concepto por primera vez, incluir una explicación breve (2-4 líneas) de:

- **Qué es**: descripción en una oración
- **Para qué sirve en este proyecto**: su rol específico aquí
- **Cómo funciona a alto nivel**: sin entrar en detalles excesivos

Ejemplos de cosas que deben explicarse la primera vez que aparecen:
- Paquetes NuGet nuevos (ej: "BCrypt.Net-Next es una librería para hashear contraseñas de forma segura...")
- Patrones de diseño (ej: "El middleware es un componente que se ejecuta en cada request antes de llegar al controller...")
- Conceptos de infraestructura (ej: "Una migración de EF Core es un archivo que describe cambios al esquema de la base de datos...")
- Configuraciones (ej: "Rate limiting es un mecanismo para limitar la cantidad de solicitudes por usuario...")

## Regla de no repetición

Si un comando o tecnología ya fue explicado en la conversación actual, no volver a explicarlo. Solo explicar conceptos nuevos.

## Idioma

Todas las explicaciones en español.
